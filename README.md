# OPERATION SANDSTORM

ブラウザで遊べるリアル志向FPS。three.js 製・ビルド不要・オフライン動作。

**▶ プレイ: https://hayashiii-ghub.github.io/fps-game/**

## 遊び方

### 起動

- 上記 GitHub Pages のURL、または
- ローカルでは `index.html` をダブルクリック（three.js はローカル同梱済み・オフライン可）
- うまく動かない場合は `python3 -m http.server` で `localhost:8000` にアクセス

### 目的 / モード

ロビーでモードを選ぶ。

- **SURVIVAL** — 砂漠の前線拠点をウェーブ防衛。WAVE 10 クリアで勝利。狙撃銃は敵狙撃兵撃破で入手。自動回復あり。
- **TEAM DEATHMATCH** — 3v3・5分・キル数勝負。装備固定（グレ2・キット2）。撃破ドロップあり。自動回復なし。

### 操作

| キー | 動作 |
| --- | --- |
| `W A S D` | 移動 |
| `Shift` | ダッシュ（射撃・ADS不可） |
| `C` | しゃがみ（集弾率向上） |
| `C + Shift` | スコープ（しゃがみ照準。スナイパー時） |
| `Space` | ジャンプ |
| 左クリック | 射撃 / グレネード投擲（構え中） |
| 右クリック | 照準（ADS：ズーム＋精度向上） |
| `R` | リロード |
| `G` | グレネード構え / 解除 |
| `F` | 応急キット使用（約2秒） |
| `Q` / `E` | 武器切替（循環・向き逆） |
| `ESC` | ポーズ（再開 / ロビーへ） |

### デバッグモード

URLにクエリを付けると検証用モードで起動する。

- `index.html?debug=1` : ポインタロックなしで即開始・状態ログ出力
- `&mode=tdm` : TDM で開始（debug時）
- `&shoot=1` : 自動で照準・射撃するボット
- `&fast=N` : 1フレームでNステップ進める時間加速シミュレーション
- `&ads=1` : ADS状態で開始

## リアル志向のポイント

### ガンプレイ

- フルオート750rpm、リコイルパターン（垂直キック＋ランダム横ブレ＋時間回復）、連射で拡がる集弾率
- 右クリックADS：FOVズーム＋レッドドットサイト照準＋感度低下＋呼吸ブレ
- 30発マガジン＋予備弾薬、リロード2.1秒、薬莢排出、マズルフラッシュ＋閃光、トレーサー
- 部位判定：ヘッドショット一撃（+150）/ 胴3発 / 四肢減衰

### 敵AI

- パトロール → 視認 or 銃声を聞きつけて索敵（距離に応じた誤差つきで発砲位置を推定）→ 交戦のステートマシン
- 反応遅延、バースト射撃、左右ストレイフ、しゃがみ、間合い管理、距離減衰する命中率
- 撃たれると赤い方向インジケータ＋ヴィネット、頭の近くを弾が通ると「crack」音

### その他

- 全音響を Web Audio API で手続き合成（銃声・足音・風・遠雷・ヒット音）
- HP自動回復、敵ドロップ（弾薬/救急キット）、キルフィード、死亡カメラ、統計付きリザルト

## 制作の流れ

1. **要件整理** — 「ブラウザですぐ遊べる」「リアル志向」「ビルド不要」の3点を優先し、依存は three.js 同梱のみの静的ファイル構成に決定
2. **ワールド生成** — Canvas 2D の手続きテクスチャ（砂・コンクリート・リブ金属・木目・迷彩）を作り、建物・コンテナ・土嚢・見張り塔・車両残骸を配置。全オブジェクトを Box3 コライダー化して移動衝突と弾道に共用
3. **ガンプレイ実装** — リコイル／スプレッド（ブルーム）／ADS／リロード／トレーサー／薬莢／マズルフラッシュ。エフェクトはすべてオブジェクトプール化
4. **敵AI実装** — patrol / search / combat のステートマシン。視線はレイキャストで遮蔽判定、聴覚は「銃声の聞こえる範囲＝拠点全域・距離比例の位置誤差」として索敵行動を駆動
5. **音響実装** — ホワイトノイズ＋フィルタ掛けで銃声・環境音を合成。距離減衰とステレオパンつき
6. **ヘッドレス自動検証** — `chrome-headless-shell` で実機レンダリングを検証。仮想時間では requestAnimationFrame が駆動しない問題に直面したため、**時間加速デバッグモード（`?fast=N`）** を実装し、射撃→キル→ウェーブ→死亡までのシミュレーションをコンソールログとスクリーンショットで確認（エラー0）
7. **バランス調整・バグ修正** — 敵の命中率・バースト間隔・反応時間を実戦ログから調整。左右を向いた際に移動方向が逆転する移動ベクトル符号バグを修正

## 使用した技術・スキル

- **three.js r128** — WebGLレンダリング、PCFSoftシャドウマップ、指数フォグ
- **Canvas 2D 手続きテクスチャ** — 画像素材ゼロで砂・金属・迷彩などを生成
- **Web Audio API 手続きサウンド** — 音声ファイルゼロで銃声・足音・環境音を合成
- **Pointer Lock API** — FPSのマウスルック
- **レイキャスト** — 弾丸の着弾判定・敵の視線（LOS）判定
- **敵AI ステートマシン** — パトロール／索敵／交戦＋聴覚モデル
- **オブジェクトプーリング** — トレーサー・パーティクル・薬莢のGC抑制
- **chrome-headless-shell による E2E 検証** — 時間加速シミュレーションとスクリーンショット目視による自動テスト手法
